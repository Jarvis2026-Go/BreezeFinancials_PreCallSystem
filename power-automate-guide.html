Phase 1 — Prep

Before You Build

Get HubSpot and Calendly ready so the flows have somewhere to write data.

1

Create custom properties in HubSpot

Go to HubSpot → Settings → Properties → Contact Properties → Create Property

Create these 5 custom contact properties: 1. Industry Single-line text 2. Monthly Revenue Single-line text 3. Pain Points Multi-line text 4. Call Time Date picker 5. Call Status Dropdown: Booked | Reminded | Showed | No-Show
2

Create deal pipeline stages in HubSpot

Go to HubSpot → Settings → Deals → Pipelines

Add these stages to your Sales Pipeline: Discovery Booked → Discovery Completed → Proposal Sent → Closed Won → Closed Lost
3

Get your Calendly API token

  • Go to Calendly → Integrations → API & Webhooks
  • Generate a Personal Access Token
  • Copy and save it
Phase 2 — Flow 1: New Booking

Fires Instantly When Someone Books

Creates the HubSpot contact, creates a deal, sends the confirmation email.

4

Open Power Automate and create a new flow

  • Go to flow.microsoft.com
  • Click "Create" → "Automated cloud flow"
  • Name it: "Breeze — New Booking"
  • Search for trigger: "Calendly — When an event is created"
  • Sign into your Calendly account when prompted
If you don't see the Calendly connector, search the connectors marketplace. It's an official connector. You may need to use the "HTTP" connector with a webhook URL as an alternative — Calendly can push events to any webhook URL.
5

Action 1: HubSpot — Create or Update Contact

  • Click "+ New step" → search "HubSpot"
  • Select "Create or update a contact (V2)"
  • Sign into your HubSpot account when prompted
  • Map the fields:
Email: [Calendly invitee email] ← this is the lookup key First Name: [Calendly invitee first name] Last Name: [Calendly invitee last name] Phone: [Calendly invitee phone] Company: [Calendly answer 1] ← business name from intake form Industry: [Calendly answer 2] ← custom property Monthly Revenue: [Calendly answer 3] ← custom property Pain Points: [Calendly answer 4] ← custom property Call Time: [Calendly event start time] Call Status: Booked Lead Source: Website — Discovery Call
"Create or Update" deduplicates on email — if this person already exists in HubSpot (e.g., from a previous inquiry), it updates their record instead of creating a duplicate.
6

Action 2: HubSpot — Create Deal

  • Add step → "HubSpot — Create a deal"
  • Map:
Deal Name: Discovery — [invitee first name] [invitee last name] Pipeline: Sales Pipeline Deal Stage: Discovery Booked Associated Contact: [Contact ID from step 5] Close Date: [Calendly event start time]
7

Action 3: Office 365 Outlook — Send Confirmation Email

  • Add step → search "Office 365 Outlook"
  • Select "Send an email (V2)"
  • It auto-connects to your Microsoft 365 tenant — no auth needed
From: no-reply@breezefinancials.com ← select from the "From" dropdown (Shared Mailbox or Send As) To: [Calendly invitee email] Subject: You're booked — here's what to expect on our call Body: (toggle to Code View, paste Email 1 HTML from emails.html) Is HTML: Yes
To send from no-reply@: In Microsoft 365 Admin Center, set up no-reply@ as a Shared Mailbox and grant your account "Send As" permission. Then in the Power Automate Outlook action, click "Show advanced options" and set the "From" field to no-reply@breezefinancials.com.
8

Save and turn on Flow 1

  • Click "Save" at the top
  • The flow is live immediately — it triggers on next Calendly booking
Phase 3 — Flow 2: Pre-Call Reminders

Runs Every Hour, Sends Timely Reminders

Checks HubSpot for upcoming calls and sends the right email at the right time.

9

Create a new Scheduled flow

  • Go to Power Automate → "Create" → "Scheduled cloud flow"
  • Name it: "Breeze — Pre-Call Reminders"
  • Set to repeat: every 1 hour
10

Action 1: HubSpot — Get Contacts

  • Add step → "HubSpot — Search for contacts"
  • Filter: Call Status equals "Booked"
  • This returns all contacts with upcoming calls who haven't been reminded yet
11

Add "Apply to each" loop

  • Add step → "Apply to each" (built-in)
  • Select the contacts array from the HubSpot step
  • Everything below runs for each contact
12

Add Condition: 24-hour reminder

  • Inside the loop, add → "Condition"
  • Configure:
If: [Contact Call Time] is greater than addHours(utcNow(), 23) AND: [Contact Call Time] is less than addHours(utcNow(), 25) AND: [Contact Call Status] equals "Booked"

If Yes:

  • → Outlook — Send email from no-reply@ (paste Email 2 HTML)
  • → HubSpot — Update contact: set Call Status to "Reminded"
Setting status to "Reminded" prevents this contact from getting the 24hr email again on the next hourly run.
13

Add Condition: Day-of reminder (2-3 hours before)

  • Add another "Condition" inside the same loop (parallel branch)
  • Configure:
If: [Contact Call Time] is greater than addHours(utcNow(), 2) AND: [Contact Call Time] is less than addHours(utcNow(), 3) AND: [Contact Call Status] equals "Reminded"

If Yes:

  • → Outlook — Send email from rodrigomedeiros@breezefinancials.com (paste Email 3 — this one is personal)
  • → (Optional) Twilio — Send SMS (see Phase 4)
14

Save and turn on Flow 2

  • Click "Save"
  • It runs automatically every hour from now on
Rescheduling works automatically: When someone reschedules in Calendly, Flow 1 fires again and updates the Call Time in HubSpot. Flow 2 then uses the new time for reminders. No duplicates because the status logic prevents re-sending.
Phase 4 — SMS (Optional)

Text Message 2-3 Hours Before

Highest-impact no-show reducer. Add to Flow 2 after the day-of email.

15

Set up Twilio

  • Sign up at twilio.com (free trial = $15.50 credit)
  • Buy a phone number (~$1.15/month)
  • Note your Account SID and Auth Token
16

Add Twilio SMS to Flow 2

  • In the day-of condition (step 13), after the Outlook email step
  • Add a Condition: "If [Contact Phone] is not empty"
  • If Yes → add step: "Twilio — Send Text Message (SMS)"
  • Connect your Twilio account
From: [your Twilio phone number] To: [HubSpot contact phone] Body: Hey [first name] — your call with Breeze Financials is in 2 hours ([time]). Join here: [zoom link] Need to reschedule? No stress: [reschedule link]
Phase 5 — Test & Go Live

Verify Everything Works

17

Test Flow 1 (instant)

  • Book a test appointment using your personal email
  • Check HubSpot: new contact with all fields? Deal created at "Discovery Booked"?
  • Check your personal inbox: confirmation email from no-reply@?
  • If something failed: go to Power Automate → Flow 1 → "Run history" to see which step errored
18

Test Flow 2 (reminders)

  • In HubSpot, edit the test contact: set Call Time to 2.5 hours from now, Call Status to "Booked"
  • Go to Flow 2 → click "Run" (manual trigger)
  • Check: did you receive the day-of email from rodrigo@?
  • Check: did you receive the SMS?
  • Check HubSpot: did status update?
19

Go live

  • Both flows should already be on
  • Flow 1 triggers automatically on bookings
  • Flow 2 runs every hour automatically
  • Monitor "Run history" in Power Automate for the first week
  • Check HubSpot daily to see contacts flowing in with full data
