Phase 1

Create Your Accounts

You need 3 free accounts before building the automation.

1

Sign up for Make.com

Go to make.com and create a free account.

  • Free tier: 1,000 operations/month
  • Each booking uses ~5-6 operations (trigger + emails + sheet + SMS)
  • That's ~160-200 bookings/month on the free plan
2

Connect Calendly

In Make.com, you'll connect your Calendly account via API key:

  • Go to Calendly → Integrations → API & Webhooks
  • Copy your Personal Access Token
  • In Make.com, when you add a Calendly module, paste this token
3

Connect Gmail (for sending emails)

Make.com will send emails from your Gmail/Google Workspace account:

  • When adding a Gmail module, Make.com prompts you to sign in
  • Authorize it to send emails on your behalf
  • Emails come from rodrigo@breezefinancials.com (your real address)
Use Google Workspace (not personal Gmail) so emails come from your business domain. This dramatically improves deliverability.
4

Create a Google Sheet for logging

Create a new Google Sheet called "Breeze Leads" with these column headers in Row 1:

A1: Date Booked B1: Name C1: Email D1: Call Time E1: Status F1: Notes
Phase 2

Build the Automation Scenario

In Make.com, a "Scenario" is your automation flow. Here's how to build it step by step.

5

Create a new Scenario

  • Click "Create a new scenario" from your Make.com dashboard
  • You'll see a blank canvas with a "+" icon in the center
6

Add trigger: Calendly → Watch Events

  • Click the "+" → search for "Calendly"
  • Select "Watch Events" (this fires when someone books)
  • Connect your Calendly account (paste the API token from Step 2)
  • Set scope to "User" and select your event type
Run the scenario once manually first — Make.com will pull a sample booking so you can see the available data fields (name, email, event time, etc.)
7

Add action: Gmail → Send Email (Confirmation)

  • Click "+" after the Calendly module → search "Gmail"
  • Select "Send an Email"
  • Configure:
To: {{Calendly invitee email}} Subject: You're booked — here's what to expect on our call Body: (paste Email 1 HTML from emails.html) Replace {{first_name}} with the Calendly "invitee name" field Replace {{date_and_time}} with the Calendly "event start time" field Replace {{zoom_link}} with the Calendly "location" field
Toggle "HTML content" to ON in the Gmail module. Then paste the full email HTML. Map the Calendly fields by clicking the field and selecting from the dropdown.
8

Add action: Google Sheets → Add Row

  • Click "+" → search "Google Sheets"
  • Select "Add a Row"
  • Select your "Breeze Leads" spreadsheet
  • Map columns:
Date Booked: {{now}} Name: {{Calendly invitee name}} Email: {{Calendly invitee email}} Call Time: {{Calendly event start time}} Status: Booked Notes: {{Calendly invitee questions/answers}}
9

Add: Sleep → Wait until 24 hours before

  • Click "+" → search "Sleep" (built-in Make.com tool)
  • Select "Sleep"
  • Set the delay using a formula:
/* Calculate: event time minus 24 hours minus current time */ Delay: {{dateDifference(addHours(Calendly.event_start; -24); now; "minutes")}}
Important: Make.com free tier has a max execution time of 5 minutes. For longer delays, use the "Schedule" approach instead: create a separate scenario that runs every hour, checks the Google Sheet for upcoming calls, and sends the email if the call is within 24 hours. This is more reliable anyway.
10

Better approach: Two separate scenarios

Instead of one long scenario with sleeps, build two scenarios:

Scenario A — "New Booking" (runs on Calendly trigger):

  • Calendly Watch Events → Send Confirmation Email → Log to Google Sheet
  • This runs instantly when someone books

Scenario B — "Pre-Call Reminders" (runs every hour on a schedule):

  • Trigger: Schedule → Every 1 hour
  • Action: Google Sheets → Search Rows where Call Time is in the future
  • Filter: If call is 23-25 hours from now → Send Email 2 (24hr reminder)
  • Filter: If call is 2-3 hours from now → Send Email 3 + SMS
This is the production-grade approach. It handles rescheduling (the Google Sheet always has the latest time), never times out, and you can see/debug every step in the sheet.
Phase 3 (Optional)

Add SMS Reminders via Twilio

Text message 2-3 hours before the call. This is the highest-impact no-show reducer.

11

Create a Twilio account

  • Go to twilio.com → Sign up (free trial includes $15 credit)
  • Buy a phone number (~$1.15/month) — this is the number texts come from
  • Note your Account SID, Auth Token, and phone number
12

Add Twilio to Scenario B

  • In your "Pre-Call Reminders" scenario, after Email 3
  • Add module: Twilio → Send an SMS
  • Connect your Twilio account
  • Configure:
From: {{your Twilio phone number}} To: {{phone from Google Sheet}} Body: Hey {{name}} — your call with Breeze Financials is in 2 hours ({{time}}). Join here: {{zoom_link}} Need to reschedule? No stress: {{reschedule_link}}
Only send SMS if the person provided a phone number. Add a Filter before the Twilio module: "Phone is not empty."
Phase 4

Test Everything

13

Test Scenario A (instant confirmation)

  • Turn on Scenario A
  • Book a test appointment through your own Calendly link using a personal email
  • Check: Did you receive the confirmation email? Did a row appear in Google Sheets?
  • If not: click "Run once" in Make.com to see which step failed
14

Test Scenario B (reminders)

  • Add a test row to your Google Sheet with a call time 2 hours from now
  • Turn on Scenario B and click "Run once"
  • Check: Did you receive the day-of email and/or SMS?
  • Adjust the time filters if needed
15

Go live

  • Set Scenario A scheduling to "Immediately" (runs as data arrives)
  • Set Scenario B scheduling to "Every 1 hour"
  • Turn both scenarios ON
  • Monitor your Google Sheet for the first few real bookings
Make.com shows execution history for every scenario run. If something fails, you'll see exactly which step broke and why. Check this weekly for the first month.
