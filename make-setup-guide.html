Phase 1 — Accounts

Connect Your Services

You already have HubSpot and Microsoft 365. You just need Make.com.

1

Sign up for Make.com

Go to make.com and create a free account.

  • Free tier: 1,000 operations/month
  • Each booking uses ~6 operations → ~160 bookings/month free
2

Get your Calendly API token

  • Go to Calendly → Integrations → API & Webhooks
  • Generate a Personal Access Token
  • Copy and save it — you'll paste this into Make.com
3

Get your HubSpot API access

  • In Make.com, when you add a HubSpot module, it uses OAuth
  • You'll sign into your HubSpot account and authorize Make.com
  • No API key needed — just click "Connect" and sign in
Before connecting, create these custom contact properties in HubSpot (Settings → Properties → Create property): Industry, Monthly Revenue, Pain Points, Call Time, Referral Source, Call Status (dropdown: Booked / Reminded / Showed / No-Show)
4

Connect Microsoft 365 email

  • In Make.com, add a "Microsoft 365 Email" module
  • Click "Create a connection" → sign in with your Microsoft account
  • Authorize Make.com to send email on behalf of no-reply@breezefinancials.com
You'll need to sign in as no-reply@breezefinancials.com (or an admin who can delegate send-as permission). If you can't sign in as that account, use the SMTP module instead with server: smtp.office365.com, port: 587, security: STARTTLS.
Phase 2 — Scenario A

Instant Booking Flow

This scenario fires the moment someone books. It creates the HubSpot contact, creates a deal, and sends the confirmation email.

5

Create Scenario A — "New Booking"

  • Click "Create a new scenario"
  • Name it "Breeze — New Booking"
6

Module 1: Calendly → Watch Events

  • Add module → search "Calendly" → select "Watch Events"
  • Connect your Calendly account (paste API token)
  • Scope: User, Event type: your discovery call
  • Click "Run once" to pull sample data
7

Module 2: HubSpot → Create/Update Contact

  • Add module → search "HubSpot CRM" → select "Create or Update a Contact"
  • Map the fields:
Email: {{Calendly.invitee_email}} First Name: {{Calendly.invitee_first_name}} Last Name: {{Calendly.invitee_last_name}} Phone: {{Calendly.invitee_phone}} (from questions) Company: {{Calendly.answer_1}} (business name from intake) Industry: {{Calendly.answer_2}} (custom property) Monthly Revenue: {{Calendly.answer_3}} (custom property) Pain Points: {{Calendly.answer_4}} (custom property) Call Time: {{Calendly.event_start_time}} (custom property) Call Status: Booked (custom property) Lead Source: Website — Discovery Call
"Create or Update" is key — if this person already exists in HubSpot (e.g., they submitted the lead magnet form first), it updates them instead of creating a duplicate. HubSpot deduplicates on email address.
8

Module 3: HubSpot → Create Deal

  • Add module → "HubSpot CRM" → "Create a Deal"
  • Map:
Deal Name: Discovery Call — {{Calendly.invitee_name}} Pipeline: Sales Pipeline (your default pipeline) Deal Stage: Discovery Booked Associated Contact: {{HubSpot contact ID from Module 2}} Close Date: {{Calendly.event_start_time}}
Create these deal stages in HubSpot first (Settings → Deals → Pipelines): Discovery Booked → Discovery Completed → Proposal Sent → Closed Won → Closed Lost
9

Module 4: Microsoft 365 → Send Confirmation Email

  • Add module → "Microsoft 365 Email" → "Send an Email"
  • Configure:
From: no-reply@breezefinancials.com To: {{Calendly.invitee_email}} Subject: You're booked — here's what to expect on our call Body type: HTML Body: (paste Email 1 from emails.html) Map {{first_name}}, {{date_and_time}}, {{zoom_link}} to Calendly fields
10

Set Scenario A scheduling

  • Click the clock icon at bottom-left of the scenario
  • Set to "Immediately" (as data arrives)
  • Turn the scenario ON
Phase 3 — Scenario B

Pre-Call Reminders

This scenario runs every hour, checks HubSpot for upcoming calls, and sends reminders at the right time.

11

Create Scenario B — "Pre-Call Reminders"

  • Create a new scenario, name it "Breeze — Pre-Call Reminders"
12

Module 1: Schedule trigger → Every 1 hour

  • Add module → search "Schedule" (built-in)
  • Set interval to 60 minutes
13

Module 2: HubSpot → Search Contacts

  • Add module → "HubSpot CRM" → "Search Contacts"
  • Filter: Call Status = "Booked" AND Call Time is in the future
  • This returns all contacts with upcoming calls
14

Add Router → two paths

  • Add a Router module (built-in) after the search
  • Path 1 — 24hr reminder: Filter where Call Time is 23-25 hours from now
  • Path 2 — Day-of reminder: Filter where Call Time is 2-3 hours from now
/* Path 1 filter (24hr reminder) */ Condition: Call Time is after {{addHours(now; 23)}} AND is before {{addHours(now; 25)}} AND Call Status = "Booked" /* Path 2 filter (day-of reminder) */ Condition: Call Time is after {{addHours(now; 2)}} AND is before {{addHours(now; 3)}} AND Call Status = "Booked"
15

Path 1 → Send 24hr Reminder Email

  • Add Microsoft 365 Email → Send
  • From: no-reply@breezefinancials.com
  • Subject: "Tomorrow: your free financial review with Breeze"
  • Body: paste Email 2 from emails.html, map HubSpot contact fields

Then add:

  • HubSpot → Update Contact: set Call Status to "Reminded"
  • This prevents sending the 24hr reminder twice
16

Path 2 → Send Day-Of Email + SMS

  • Add Microsoft 365 Email → Send
  • From: rodrigo@breezefinancials.com (this one should feel personal)
  • Subject: "See you at {{time}} — here's your Zoom link"
  • Body: paste Email 3 from emails.html

Then optionally add Twilio SMS (see Phase 4).

17

Set Scenario B scheduling

  • Set to run every 60 minutes
  • Turn the scenario ON
Why this works with rescheduling: If someone reschedules in Calendly, Scenario A fires again with the new time, updating the HubSpot contact's Call Time. When Scenario B checks next hour, it uses the new time automatically. No duplicate reminders.
Phase 4 — SMS (Optional)

Add Text Message Reminder

A text 2-3 hours before the call is the single highest-impact no-show reducer.

18

Set up Twilio

  • Sign up at twilio.com (free trial = $15 credit)
  • Buy a phone number (~$1.15/month)
  • Note your Account SID, Auth Token, and phone number
19

Add to Scenario B, Path 2

  • After the day-of email, add: Twilio → Send SMS
  • Add a Filter before it: "Phone is not empty"
  • Configure:
From: {{your Twilio number}} To: {{HubSpot contact phone}} Body: Hey {{first_name}} — your call with Breeze Financials is in 2 hours ({{time}}). Join here: {{zoom_link}} Need to reschedule? No stress: {{reschedule_link}}
Phase 5 — Test & Go Live

Test Everything

20

Test Scenario A

  • Turn on Scenario A
  • Book a test appointment with your personal email
  • Check: Did HubSpot get a new contact with all fields populated?
  • Check: Did a deal appear in your pipeline at "Discovery Booked"?
  • Check: Did you receive the confirmation email from no-reply@?
21

Test Scenario B

  • In HubSpot, set the test contact's Call Time to 2.5 hours from now
  • Set Call Status to "Booked"
  • Turn on Scenario B and click "Run once"
  • Check: Did you receive the day-of email from rodrigo@?
  • Check: Did you receive the SMS (if Twilio is configured)?
22

Go live

  • Scenario A: set to "Immediately"
  • Scenario B: set to every 60 minutes
  • Both scenarios ON
  • Monitor HubSpot + Make.com execution history for the first week
